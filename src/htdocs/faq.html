How should I export an SVG?

Best practices are also in the upload form, and repeated below. You can also check this image.

Exporting from Illustrator: You should use the "Save as" function in Illustrator to save your SVG. The proper settings should be the defaults, and are listed below:

  • First of all, remove any extra data or images you don't want to appear!
  • Under Fonts -> Type make sure "SVG" is selected
  • Under Options -> Image Location select "Embed"
  • Under Advanced Options -> CSS Properties select "Style Elements"
  • Under Advanced Options -> Decimal Places set to 1
  • Select the checkboxes "Output fewer <tspan> elements," "Use <textPath> element for Text on Path, and "Responsive."

When do I want to use an SVG blurb as opposed to an image?

Pretty much always. Using SVGs result in crisper images, smaller file sizes, and better display across devices compared to static images.

If you have a particularly large or detailed SVG, however, you may be off exporting an image, which will result in a smaller file size.

Can I use SVG blurbs for lead art?

Yes! Just add the blurb as the first related item in P2P. The only caveat is that this is the one case where you will need a 16:9 aspect ratio in your SVG.

What fonts can I use in my SVG?

To reduce the number of fonts we have to serve to our users, please keep the fonts used in your graphics to Benton Gothic Regular, Benton Gothic Bold, and Benton Gothic Black.

I exported the SVG. Now what?

Your SVG will save as a blurb in working in P2P. For now, you'll have to go and turn the blurb live manually, though this may change in the future.

If you're working on a story in SNAP, add the slug of the blurb to a P2P embed item. Not that the SVG will not show up in the story -- SNAP cannot read the contents of the blurb, so it will not display in SNAP, but will show in P2P preview.

I don't see my blurb in SNAP. What happened?

Note that SNAP can't read the contents of a blurb, so you'll just see an empty space in the story editing screen in SNAP. You will be able to see the SVG if you preview in P2P, however.

My type looks weird in the SVG in P2P. What gives?

Most likely, one of two things happened:

  1. There were fonts other than Benton Gothic Regular, Benton Gothic Bold, and Benton Gothic Black included in the SVG.
  2. The SVG was pasted directly into the P2P blurb, without using the uploader. The uploader performs a number of fixes to get the SVG to display properly in P2P, so please do not paste directly into P2P.
